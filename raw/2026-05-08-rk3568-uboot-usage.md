## Raw Notes

这篇文章主要介绍 RK3568 开发板上 U-Boot 的使用方法。

U-Boot 是 Linux 系统启动前运行的 bootloader。芯片上电后会先运行 bootloader，完成 DDR 等外设初始化，然后把 Linux 内核从 NAND、NOR Flash、SD、eMMC 等存储介质拷贝到 DDR 中，最后启动 Linux 内核。

文章使用的是正点原子 ATK-DLRK3568 开发板。开发板出厂系统已经烧写了 U-Boot，因此可以直接通过串口进入 U-Boot 命令行体验相关命令。

## Key Points

### 1. U-Boot 是什么

- U-Boot 全称是 Universal Boot Loader。
- 它是遵循 GPL 协议的开源 bootloader。
- 它本质上是一个裸机程序，可以理解为一个功能比较完整的裸机综合例程。
- 当前 U-Boot 已经支持 LCD、网络、USB 等高级功能。
- 在 Linux 启动链路中，U-Boot 的核心任务是启动 Linux 内核。

### 2. RK3568 的 U-Boot 来源

- 一般不会直接使用 U-Boot 官方源码。
- 官方源码通常提供给半导体厂商，厂商会基于官方源码移植自家芯片支持。
- 瑞芯微 SDK 中提供了适配 RK3568 的 U-Boot。
- 正点原子的 SDK 进一步针对 ATK-DLRK3568 开发板做了适配。
- 如果使用瑞芯微原厂 SDK，可能还需要根据自己的板子做移植。

### 3. 进入 U-Boot 命令行

操作流程：

1. 使用 USB Type-C 线连接开发板 UART 接口和电脑。
2. 打开 MobaXterm 或其他串口终端。
3. 设置串口参数。
4. 复位开发板。
5. 出现 `Hit key to stop autoboot('CTRL+C'):` 倒计时时按下 `CTRL+C`。
6. 进入 U-Boot 命令行模式。

如果倒计时结束前没有按下 `CTRL+C`，U-Boot 会继续启动 Linux 内核。

### 4. U-Boot 启动日志可以观察的信息

启动日志中通常可以看到：

- U-Boot 版本号
- 编译时间
- 板子信息
- DDR 大小
- eMMC 启动设备信息
- PMIC 芯片信息，例如 RK809
- DRM / 显示相关信息
- RK3568 芯片时钟信息

### 5. 查询类命令

常用查询命令：

```bash
help
?
bdinfo
printenv
version
````

说明：

- `help` 或 `?`：查看当前 U-Boot 支持的命令。
    
- `bdinfo`：查看板级信息，如 DRAM 起始地址、大小、启动参数地址、波特率、栈指针等。
    
- `printenv`：查看环境变量。
    
- `version`：查看 U-Boot 版本。
    

### 6. 环境变量操作

相关命令：

```bash
setenv
saveenv
printenv
```

示例：修改启动倒计时为 3 秒。

```bash
setenv bootdelay 3
saveenv
```

关键理解：

- `setenv` 修改的是 DRAM 中的环境变量。
    
- 如果不执行 `saveenv`，重启后修改不会保存。
    
- `saveenv` 会把环境变量保存到外部 Flash / eMMC 中。
    

新建环境变量：

```bash
setenv author 'alientek'
saveenv
```

删除环境变量：

```bash
setenv author
saveenv
```

### 7. 内存操作命令

常用命令：

```bash
md
nm
mm
mw
cp
cmp
```

说明：

- `md`：显示内存数据。
    
- `nm`：修改指定地址的内存值，地址不会自增。
    
- `mm`：修改内存值，地址会自动递增。
    
- `mw`：用指定数据填充一段内存。
    
- `cp`：拷贝内存数据。
    
- `cmp`：比较两段内存数据是否一致。
    

注意：

U-Boot 命令中的数字通常按十六进制理解，不是十进制。

示例：

```bash
md.b 8300000 10
md.w 8300000 10
md.l 8300000 10
```

其中：

- `.b` 表示 byte，1 字节。
    
- `.w` 表示 word，2 字节。
    
- `.l` 表示 long，4 字节。
    

### 8. 网络相关命令

U-Boot 支持网络命令，常见命令包括：

```bash
ping
dhcp
nfs
tftpboot
```

常见网络环境变量：

```bash
setenv ipaddr 192.168.5.4
setenv gatewayip 192.168.5.1
setenv netmask 255.255.255.0
setenv serverip 192.168.5.2
saveenv
```

说明：

- `ipaddr`：开发板 IP。
    
- `serverip`：服务器 IP，通常是 Ubuntu 主机。
    
- `gatewayip`：网关。
    
- `netmask`：子网掩码。
    
- `ethaddr`：MAC 地址，有些 U-Boot 默认禁止修改。
    

注意：

U-Boot 可以 ping 其他机器，但其他机器通常不能 ping U-Boot，因为 U-Boot 没有处理来自外部的 ping 请求。

### 9. NFS 下载

NFS 用于通过网络文件系统从 Ubuntu 主机下载文件到开发板 DRAM。

Ubuntu 安装 NFS 服务：

```bash
sudo apt-get install nfs-kernel-server rpcbind
```

配置 `/etc/exports` 示例：

```bash
/home/xhj/linux/nfs *(rw,sync,no_root_squash)
```

重启 NFS 服务：

```bash
sudo /etc/init.d/nfs-kernel-server restart
```

U-Boot 中使用 NFS 下载文件：

```bash
nfs 30000000 192.168.5.2:/home/xhj/linux/nfs/boot.img
```

含义：

- `30000000`：文件下载到 DRAM 的地址。
    
- `192.168.5.2`：Ubuntu 主机 IP。
    
- `/home/xhj/linux/nfs/boot.img`：主机上的文件路径。
    

### 10. TFTP 下载

TFTP 和 NFS 类似，也用于通过网络把文件下载到 DRAM。

Ubuntu 安装 TFTP：

```bash
sudo apt-get install tftp-hpa tftpd-hpa
sudo apt-get install xinetd
```

创建 TFTP 目录：

```bash
mkdir /home/xhj/linux/tftpboot
chmod 777 /home/xhj/linux/tftpboot
```

U-Boot 中下载：

```bash
tftp 30000000 boot.img
```

和 NFS 的区别：

- NFS 需要写完整主机路径。
    
- TFTP 只需要写文件名，因为文件默认从 TFTP 根目录读取。
    

### 11. MMC / eMMC / SD 卡命令

U-Boot 中常用 `mmc` 命令操作 eMMC 和 SD 卡。

常用命令：

```bash
mmc info
mmc rescan
mmc list
mmc dev
mmc part
mmc read
```

说明：

- `mmc info`：查看当前 MMC 设备信息。
    
- `mmc rescan`：重新扫描 MMC 设备。
    
- `mmc list`：列出所有 MMC 设备。
    
- `mmc dev`：切换当前 MMC 设备。
    
- `mmc part`：查看分区。
    
- `mmc read`：从 MMC 读取数据到 DRAM。
    

示例：

```bash
mmc dev 0
mmc part
mmc read 8300000 400 10
```

含义：

- 切换到 eMMC。
    
- 查看 eMMC 分区。
    
- 从 eMMC 的 0x400 块开始读取 0x10 个块到 DRAM 地址 `0x8300000`。
    

### 12. EXT4 文件系统命令

RK3568 系统镜像通常使用 EXT4 文件系统，常用命令：

```bash
ext4ls
ext4load
ext4write
```

示例：查看 eMMC 第 6 分区文件。

```bash
ext4ls mmc 0:6
```

示例：加载文件到 DRAM。

```bash
ext4load mmc 0:6 8300000 linuxrc
```

### 13. BOOT 相关命令

U-Boot 的核心任务是引导 Linux。

RK3568 中常用：

```bash
boot_fit
boot
bootd
```

RK3568 的系统烧写文件通常是 `boot.img`，Linux 内核 Image 和设备树文件都打包在 `boot.img` 里，因此不能简单使用传统的 `bootm` 或 `bootz`，而是使用 `boot_fit`。

从 eMMC 默认启动：

```bash
boot_fit
```

从 TFTP 下载后启动：

```bash
tftp E9EF7000 boot.img
boot_fit E9EF7000
```

`boot` 和 `bootd` 会执行环境变量 `bootcmd` 中定义的启动命令。RK3568 中 `bootcmd` 默认会调用 `boot_fit`，所以倒计时结束后会自动启动 Linux。
